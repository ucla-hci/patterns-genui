# Key Points

### Key elements to include when describing a design pattern

- **Teaser**: a leading visual illustration, either a representative example or a sketch of the general pattern
- **Problem**: what problem does this design pattern facilitate solving?
- **Use when**: when to use (and not)
- **Principle**: any theoretical or empirical or design principles underpinning this pattern
- **Solution**: describe the detail solutions
- **Examples**: more examples for references

### Other things to include

- Variations of the solution, e.g., for Undo, there is Selective Undo
