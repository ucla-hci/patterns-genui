# One Sentence
Proposes a collection of mobile UI design patterns organized into problem areas, presenting six patterns in detail as examples of the broader collection.

---

# More Sentences
Nilsson derives patterns from experience with mobile application development, organizing them by "problem area" rather than screen type or interaction concern. Six representative patterns from different problem areas are described in full (name, problem, context, solution, consequences). The journal context makes this the most rigorous academic mobile pattern catalog predating Neil (2014). Limitations: no formal evaluation, predates modern smartphones (iOS/Android era), and the full catalog is not published.

---

# Key Points

### Problem-area organization
Patterns grouped by the type of design problem they solve, not by screen or interaction type — an alternative taxonomy to Neil's 11 categories.

### Pre-touchscreen era
Published in 2009, shortly after iPhone launch — patterns may not reflect touch-first design. A gap that post-2012 work must address.

### Pattern template includes "consequences"
Nilsson's template explicitly includes consequences (trade-offs of applying the pattern) — closer to Alexander's forces/resolution structure than most catalogs.

# Other Notes
The most academically rigorous early mobile pattern paper. Complementary to Neil (2014): Nilsson is earlier, more structured, smaller scope; Neil is later, more comprehensive, more visual.

# Take-Away
- Establishes that mobile pattern catalogs can be published in peer-reviewed journals — raising the bar for PatternGenUI's pattern library publication.
- The "consequences" field in the pattern template is worth adopting in PatternGenUI's spec language.
