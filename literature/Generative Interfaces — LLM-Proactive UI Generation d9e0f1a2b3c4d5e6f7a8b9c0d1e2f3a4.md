# One Sentence
Proposes a paradigm in which LLMs proactively generate task-specific UIs for their own outputs (rather than text responses), using structured representations (interaction flows + FSMs) and iterative refinement.

---

# More Sentences
The structured intermediates are high-level interaction flows (user trajectories) and low-level FSMs (component behavior). The focus is LLMs generating their own response interfaces, not a designer tool for UI code generation. A human preference evaluation showed 72% improvement over chat-based baselines.

---

# Take-Away
- Different use case: LLM-as-UI-generator for conversational outputs. Not a designer tool. Not a prior art risk for PatternGenUI's claim.
