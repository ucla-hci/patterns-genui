# One Sentence
Catalogs ~100 named UI patterns with visual examples covering desktop, web, and mobile, organized by user goal and screen context.

---

# More Sentences
Tidwell presents patterns organized into chapters by interaction concern (e.g., "Getting Input," "Showing Complex Data," "Navigation"), each with name, thumbnail, usage context, and annotated screenshots. The 3rd edition adds mobile and social-media patterns. It is arguably the most widely used practitioner pattern reference in UX design. Patterns are expert-derived; no formal validation study exists.

---

# Key Points

### Organization by interaction concern, not domain
Chapters like "Organizing the Page," "Actions and Commands," "Showing Complex Data" cut across mobile/web/desktop — domain-neutral.

### Pattern template
Each entry: name, thumbnail example, "What" (the pattern), "Use when," "Why," "How," annotated screenshots.

### Breadth over depth
~100 patterns but relatively shallow descriptions; no formal context/problem/solution/forces structure per Alexander.

# Other Notes
The 3rd edition (2019) is current. Most widely cited practitioner catalog. Closest analog to what PatternGenUI's pattern library might look like for designers — but lacks machine-readable structure.

# Take-Away
- Establishes the practitioner gold standard: rich, visual, named patterns that designers actually use.
- PatternGenUI's pattern descriptions should be at least as readable as Tidwell while adding a structured (retrievable) schema.
